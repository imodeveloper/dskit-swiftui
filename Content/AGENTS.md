# Agent Instructions

## Purpose
- Repository documentation and visual assets used by README and component docs.

## Notes
- `Views.md`, `Screens.md`, `Layout-in-DSKit.md`, `Appearance-in-DSKit.md`, and `Documentation.md` are referenced in README.
- `Documentation.md` is the hand-maintained generated-docs workflow guide.
- `Screen-Construction.md` is the hand-maintained guide for building full DSKit screens from DSKitExplorer patterns.
- `docs/` contains hand-maintained workflow, quality, planning, design-principle, and execution-plan docs.
- `Views.md` is the generated component table of contents.
- `Views/*.md` and `Views/UsageIndex.md` are generated by `Scripts/documentation_generator.sh`.
- `Screens.md` is the generated DSKitExplorer screen table of contents.
- `Screens/*.md` is generated by `Scripts/documentation_generator.sh`.
- `Screens/Frames/*.framed.png` is generated by `Scripts/documentation_generator.sh` from screen snapshot PNGs.
- `Screens/Groups/*.strip.png` is generated by `Scripts/documentation_generator.sh` from framed screen preview PNGs.
- Every generated view page requires an exact preview image in `DSKitTests/__Snapshots__/DSKitTests/<Component>.snapshot.png`.
- Every generated screen page requires at least one snapshot image in `DSKitExplorerTests/__Snapshots__/DSKitExplorerTests/<Screen>.snapshot.png` or a numbered variant such as `<Screen>_0.snapshot.png`.
- Do not hand-edit generated component pages, screen pages, screen frame PNGs, or screen strip PNGs; update Swift source, snapshots, or the generator.
